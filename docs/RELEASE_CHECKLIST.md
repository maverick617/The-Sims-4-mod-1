# Release Checklist

Use this before uploading a GitHub Release.

- [ ] The `.package` file has been tested in game.
- [ ] The item appears in the expected CAS or Build/Buy category.
- [ ] The item name and description are correct.
- [ ] Swatches display correctly.
- [ ] No unrelated test files are included.
- [ ] Asset credits are up to date.
- [ ] Screenshots are included in the release.
- [ ] The release notes mention the supported game patch version, if known.
